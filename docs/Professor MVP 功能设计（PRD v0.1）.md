### 0. 文档信息

- 产品：Professor（占位名）
- 版本：MVP v0.1
- 目标平台：Web（桌面浏览器为主，后续可扩展 iOS 客户端）
- 最核心能力：
  - 从 B 站「系列视频」一键导入一批视频；
  - 将视频放入不同 Notebook；
  - 对单个 / 多个 / 全部视频生成中/深度总结（含章节）；
  - 基于这些内容 + 大模型世界知识进行问答；
  - 用户可以像 NotebookLM 一样做笔记。

### 1. 产品定位（针对 MVP）

一句话：

Professor = 面向自己学习的视频知识整理工具，专注 B 站和 YouTube 视频 → Notebook → 深度总结 + 问答。

MVP 只做三件事：

1. 高效「批量拉取」B 站和 YouTube 视频，（支持 B 站系列视频和 YouTube 的视频分组，解决一个个粘贴的痛点）。
2. 把这些视频「组织进 Notebook」并生成可替代观看的文字总结。
3. 提供一个「基于 Notebook 的深度问答 + 笔记空间」。

### 2. 目标用户 & 使用场景（MVP 版）

#### 2.1 目标用户（第一阶段就是我自己 + 一小撮类似用户）

- 重度通过 B 站/视频学习某个专题的人，比如以下专题：
  - 独立开发 / indie 创业生态
  - 编程教程、技术栈学习
  - 商业/产品/设计课程
  - 其它长系列课程

#### 2.2 核心使用场景

1. 场景 A：导入一个 B 站系列
   - 用户在 B 站发现一个「合集/系列/播放列表」。
   - 粘贴系列链接 → FlashNote 自动拉取该系列全部视频的列表。
   - 用户勾选需要的视频 → 选择已有 Notebook 或新建 Notebook → 批量导入。
2. 场景 B：看总结，不看视频（为主流程）
   - Notebook 中显示导入的视频列表。
   - 用户点击某个视频 → 看到「中/深度总结 + 章节大纲」，基本可以不再看视频。
   - 需要在视频文件上标注标签（根据总结的章节），当需要回看画面/图示的时候，可以标签跳到原视频的制定位置。
3. 场景 C：基于若干视频/整个 Notebook 提问
   - 用户学了一段时间以后，希望基于「某几个视频」或「整个 Notebook」提问：
     - 比如：
       - 「这些独立开发者在支付选型上的常见路径和坑是什么？」
       - 「帮我总结一下这个系列里提到的开发者收入模型。」
   - 系统使用 Notebook 的内容 + 大模型世界知识生成深度回答。

（以上场景也支持 YouTube 视频）

4. 场景 D：记笔记
   - 用户在看某个视频的总结时，随手记录补充思考/感想（类似 NotebookLM 的 note 面板）。
   - 笔记与视频/Notebook 关联，方便之后复盘。

### 3. 产品范围（MVP：做什么 & 不做什么）

#### 3.1 做什么（In Scope）

1. 账号 & 基础 Notebook 管理（简单版）
   - 用户登录（基于现有的官网登录，集成登录，见 XiPiLabs 产品单点登录集成方案.md）。
   - Notebook 的创建、重命名、删除。
   - 视频与 Notebook 的关联管理（一个视频至少属于一个 Notebook）。
2. B 站导入（系列优先）
   - 支持输入以下类型之一：
     - 单个视频链接
     - 系列 / 合集 / 播放列表链接（一次返回多个视频）
   - 展示该链接下视频列表：标题、时长、封面、分 P 信息等。
   - 支持用户：
     - 全选 / 反选
     - 自定义勾选部分视频
     - 选择导入目标 Notebook（可新建）
3. 视频内容抓取 & 转写
   - 利用现有的 B 站解析工具（开源基础能力）：
     - 优先读取现有字幕 / CC（如有）。
     - 无字幕时，下载音频并用自有 ASR（如 Whisper）转写。
   - 转写结果：保存为文本（后端存储即可，前端不一定全部展示）。
4. 总结生成（中度 / 深度，总结模式可配置）
   - 系统级 / 用户级「默认总结模式」设置：
     - 仅中度总结
     - 仅深度总结
     - 两者都生成（MVP 可以只支持前两种）
   - 对每个视频：
     - 中度总结：
       - 1–3 段整体描述 + 5–10 条要点列表。
     - 深度总结：
       - 按章节结构组织，含小标题 + 每节详细说明。
       - 总体长度足以「替代观看」为目标。
   - 两个总结模式在代码层共用 pipeline，只是提示词不同。
5. 章节大纲（含时间戳）
   - 对转写内容进行分段：
     - 按语义 + 时间划分章节（例如 3–10 个章节）。
   - 每个章节包含：
     - 起始时间戳（如 03:21）
     - 章节标题
     - 1–3 句章节小结。
   - 前端支持：点击章节 → 打开 B 站对应时间点（新标签页 / 悬浮播放器）。
6. 问答（单视频 / 多视频 / 整个 Notebook）
   - 用户在 Notebook 页面中：
     - 可以勾选多个视频后，点击「基于所选视频提问」。
     - 也可以选择「使用 Notebook 全部视频提问」。
   - 问答方式：
     - 问题输入框 + 聊天记录区域（类似 Chat）。
     - 回答的风格要求：

优先利用 Notebook / 视频内容：

告诉模型：“这些是用户自己整理的视频总结 / 笔记，是当前问题的主要背景，请优先参考。”

允许充分使用世界知识：

明确写：  
“你可以结合自身的通用知识，补充解释相关概念、给出更系统的背景和延伸。”

输出结构化：

要求模型：

使用小标题；

分段；

适当用列表总结关键点。

    - 后端实现策略（高层）：
        * 对每个视频存储「深度总结」，并以此做 embedding。
        * Notebook 级问答先检索相关视频，再检索对应总结片段作为上下文传给大模型。
        * Prompt 明确允许模型补充世界知识。

7. 笔记功能（仿 NotebookLM 的最小版本）
   - 在「视频详情页面（总结视图）」右侧或底部提供「笔记区域」。
   - 功能：
     - 为当前视频添加多条笔记（纯文本即可）。
     - 支持简单的标题 + 正文（可选）。
   - 在 Notebook 层面：
     - 可以查看所有视频的笔记列表（按视频/时间分组即可）。

#### 3.2 不做什么（Out of Scope for MVP）

- 不做 iOS 客户端（先完全走 Web，之后共用后端接口）。
- 不做复杂用户系统（团队、分享、协作等）。
- 不做学习卡片/测验题（放到下一阶段）。
- 不做跨 Notebook 分析、自动学习路径推荐。
- 不做多平台支持（暂不支持 YouTube、Podcast 等，先搞定 B 站）。

### 4. 关键页面 & 交互流程（MVP）

#### 4.1 页面列表

1. 登录页（使用官网登录，单点集成）
2. Notebook 列表页
3. Notebook 详情页
   - 顶部：Notebook 基本信息（名称、描述）
   - 左侧：视频列表（可勾选）
   - 中部：Notebook 级问答入口
   - 右侧：其他功能卡片（参照 google 的 notebooklm）
4. B 站导入页（或弹窗）
   - 输入链接 → 展示视频列表 → 勾选 → 选择 Notebook → 导入
5. 视频详情页
   - 视频基本信息（标题、时长、B 站链接）
   - 总结区（中度/深度，按用户偏好显示为默认）
   - 章节大纲区（含时间戳）
   - 问答区（针对该视频提问）
   - 笔记区（笔记列表 + 新建笔记）

#### 4.2 核心流程 1：导入 B 站系列并建 Notebook

1. 用户在 Notebook 列表页点击「新建 Notebook」
   - 输入名称（例如「独立开发者生态」）。
2. 进入 Notebook 详情页，点击「从 B 站导入」
3. 粘贴 B 站系列/合集链接 → 点击「获取视频列表」
4. 后端解析链接，返回视频列表（标题、时长、封面等）
5. 前端展示列表，支持：
   - 全选 / 反选
   - 单个勾选
6. 用户确认 → 点击「导入到当前 Notebook」
7. 后端为被选视频创建记录：
   - 存基础元数据（标题、链接、时长）
   - 加入队列：执行转写 + 总结（可以异步）
8. Notebook 页面展示：
   - 新加入的视频条目，状态为「处理中」→ 完成后变为「已总结」。

---

#### 4.3 核心流程 2：查看深度总结 & 章节大纲

1. 用户进入 Notebook 详情页。
2. 点击某个已完成的视频条目 → 跳转至视频详情页。
3. 视频详情页结构：
   - 标题 + 基本信息 + B 站跳转按钮
   - 「总结模式切换」：
     - Tab 或下拉：中度总结 / 深度总结
   - 章节大纲：按时间顺序列出章节，点击可打开 B 站对应时间。
   - 笔记区域：查看/创建笔记。

---

#### 4.4 核心流程 3：Notebook 级问答（单/多/全部视频）

1. 用户在 Notebook 详情页勾选若干视频（或不勾选，默认「全部」）。
2. 点击「开始问答」按钮。
3. 右侧/底部 Chat 区出现当前问答会话：
   - 显示本次问答的作用范围（例如：“基于 5 个选中视频”或“基于整个 Notebook”）。
4. 用户输入问题 → 回车发送。
5. 后端流程（高层）：
   - 根据问题 + 目标范围，检索相关视频总结片段；
   - 构造上下文 + 问题给大模型；
   - 模型回答，允许补充世界知识。
6. 前端展示回答：
   - 尽量结构化；
   - 可以简单显示「参考了 X 个视频」（不必展示全部引用细节）。

---

#### 4.5 核心流程 4：视频级问答 + 笔记

1. 在视频详情页，用户在问答区直接提问：
   - 问题默认为「基于当前视频 + 模型世界知识」。
2. 回答展示在视频页面的问答历史中。
3. 用户在阅读总结时，可以随时在笔记区添加一条笔记：
   - 笔记内容：标题（可选）+ 正文（必填）
   - 自动记录创建时间。

### 5. 功能需求清单（按模块整理）

#### 5.1 Notebook 管理模块

- [MVP] 新建 Notebook（名称 + 可选描述）。
- [MVP] 重命名 / 删除 Notebook。
- [MVP] 查看 Notebook 列表。
- [MVP] 在 Notebook 中查看视频列表。

#### 5.2 B 站导入模块

- [MVP] 输入单个视频链接导入。
- [MVP] 输入系列/合集链接：
  - 解析所有子视频信息；
  - 展示为可选列表；
  - 批量勾选导入。
- [MVP] 支持将导入视频关联到当前 Notebook。
- [可选后续] 支持从「UP 主主页」导入某一列表/合集。

#### 5.3 转写 & 总结模块

- [MVP] 对每个视频：
  - 拉取字幕或音频 → 转写。
  - 保存转写文本（后台）。
  - 基于转写生成：
    - 中度总结（可选）
    - 深度总结（可选）
    - 章节大纲（含时间戳）
- [MVP] 用户偏好设置：
  - 默认只生成「中度总结」或「深度总结」。

#### 5.4 问答模块

- [MVP] 视频级问答：
  - 提问范围：当前视频。
  - 回答可用世界知识。
- [MVP] Notebook 级问答：
  - 范围选择：勾选视频 / 全部视频。
  - 后端检索相关视频总结片段，构造上下文。
- [后续] 保留问答会话历史并可搜索。

#### 5.5 笔记模块

- [MVP] 视频级笔记：
  - 为当前视频新增笔记（纯文本）。
  - 列表展示、按时间排序。
- [MVP] Notebook 级笔记视图：
  - 汇总展示所有笔记，按视频分组。

### 6. 非功能需求（MVP 简版）

- 性能：
  - 导入后转写与总结可以是异步任务，不要求实时完成。
  - 页面需要有状态提示（排队中 / 处理中 / 已完成）。
- 成本控制：
  - 对每个视频只做 1 次转写 + 总结，后续问答重用这些结果。
  - Notebook 级问答只检索相关视频，不直接把所有文本一次性塞进模型。
- 隐私：
  - 数据默认只存你自己账户下。
  - 后续对外用户时需增加隐私条款和数据删除功能。

### 7. 后续可以扩展的方向（不属于 MVP，但提前预留思路）
